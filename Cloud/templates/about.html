About This App

This demo showcases how a Flask web application can be containerized using Docker and deployed to Google Cloud Run — a fully managed serverless platform that automatically scales your containers based on incoming traffic.

How It Works

  • 1. Flask Application: A lightweight Python web app serves the frontend and backend logic.
  • 2. Docker Containerization: The app is packaged into a Docker image with all its dependencies defined in Dockerfile.
  • 3. Deployment: The container image is pushed to Google Container Registry (GCR) or Artifact Registry.
  • 4. Cloud Run: The image is deployed on Cloud Run, which handles scalability, load balancing, and HTTPS automatically.
  • 5. Continuous Integration: With Cloud Build or GitHub Actions, you can automate deployments directly from source control.

Why Cloud Run?

Cloud Run eliminates the need to manage infrastructure. It offers auto-scaling, pay-per-use billing, and supports any language or framework that runs in a container — making it ideal for microservices and modern cloud-native workloads.
